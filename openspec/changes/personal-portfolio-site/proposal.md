# 个人作品展示网站提案

## 动机

作为一名 Golang 工程师，希望通过接外包项目来拓展收入来源。目前缺乏有效的展示渠道（如闲鱼、小红书等平台），需要一个专业的个人作品网站来展示技术能力、项目经验和接单信息。该网站将成为个人品牌的核心载体，帮助潜在客户快速了解服务内容和过往成果。

## 建设内容

本项目将构建一个单页面个人作品展示网站，包含以下核心模块：

**首页与个人简介**——展示工程师的核心技能、专业背景和接单状态。采用视觉层次分明的排版，突出 Golang 相关的技术栈（微服务、分布式系统、云原生等）。

**项目作品集**——以卡片形式展示过往项目，每个项目包含技术方案、核心功能、开发周期和客户评价。支持项目分类筛选（Web 开发、后端架构、API 开发等）。

**服务与定价**——清晰列出可承接的项目类型及价格区间，包括：Web 后端开发、API 接口设计、微服务架构、代码重构与优化、技术咨询等。

**联系方式**——提供多种联系方式（微信、邮箱、GitHub），并集成在线表单便于客户直接提交需求。

## 新增能力

- **portfolio-homepage**：个人作品展示首页，包含个人简介、技能标签和服务介绍
- **portfolio-projects**：项目作品集模块，支持项目展示、分类筛选和详情展开
- **portfolio-services**：服务定价页面，列出可承接的项目类型和报价
- **portfolio-contact**：联系表单和社交链接集成

## 影响范围

本项目为前端静态网站，不涉及后端服务部署。主要影响包括：需要设计响应式布局以适配桌面端和移动端；集成第三方表单服务（如 Formspree）处理联系请求；使用 CDN 托管静态资源以保证访问速度。
