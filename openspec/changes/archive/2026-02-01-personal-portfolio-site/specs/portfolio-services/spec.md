# 服务定价模块规范

## 概述

服务定价模块是个人作品展示网站的重要转化页面，面向潜在客户清晰传达可承接的项目类型、服务范围和价格区间。该模块通过结构化的服务卡片展示各项服务内容，帮助客户快速评估合作价值和预算匹配度。设计目标是消除客户对价格的不确定性，建立透明的报价机制，降低商务沟通成本，提升咨询转化率。

模块采用服务卡片网格布局，每张卡片代表一类服务项目，包含服务名称、核心功能、适用场景、价格区间和常见问题解答。页面同时展示合作流程说明和客户见证区域，增强专业可信度。服务定价信息应保持适度灵活性，避免过于僵化的报价导致优质客户流失，同时提供足够的价格锚点帮助客户建立合理预期。

## 功能需求

### 服务卡片

服务卡片以垂直布局展示单项服务的完整信息。卡片顶部显示服务图标和名称，服务图标使用与技能标签一致的配色风格。服务名称下方以简洁文字描述服务的核心价值主张，如「快速构建高性能 API 接口」。功能列表以复选框或项目符号形式展示服务包含的具体内容，每项包含功能名称和简短说明。

适用场景区域说明该服务的典型客户需求和项目规模，如「初创公司 MVP 开发」「现有系统 API 扩展」「微服务模块独立开发」等。价格区间以「起步价 - 封顶价」或「xxx 元起」的形式展示，如「5000 - 20000 元」或「3000 元起」。价格说明以小字标注价格区间对应的项目规模和交付周期。卡片底部设置「预约咨询」按钮，点击后跳转至联系模块或展开咨询表单。

服务卡片支持三种状态：正常显示、悬停高亮、已收藏（可选）。悬停时卡片轻微上浮并添加阴影，颜色方案保持不变。部分服务可标记为「热门」或「新上线」，使用角标或标签突出显示。

### 服务分类

服务项目按类型划分为五个主要类别：Web 后端开发、API 接口设计、微服务架构、代码优化咨询和技术培训。每个类别有独立的入口展示区域，页面顶部通过标签导航快速切换。分类切换采用淡入淡出动画，避免生硬的页面刷新。

「Web 后端开发」类别包含常规 Web 应用开发、管理后台开发、CRM/ERP 系统开发等服务项。「API 接口设计」类别包含 RESTful API 开发、GraphQL 接口设计、第三方 API 集成、API 文档编写等服务项。「微服务架构」类别包含微服务拆分设计、服务治理方案、容器化部署、云原生架构等服务项。「代码优化咨询」类别包含性能调优分析、代码重构方案、技术债务清理、架构评审咨询等服务项。「技术培训」类别包含 Go 语言入门培训、后端架构设计培训、团队技术指导等服务项。

### 合作流程

合作流程区域以时间线或步骤图形式展示标准合作流程。流程步骤包括：需求沟通（免费，1 至 3 天）、方案确认（免费，2 至 5 天）、合同签订、启动开发、阶段交付、项目验收、售后支持。每个步骤有图标、标题和一句话说明。已完成步骤显示绿色对勾，进行中步骤高亮显示， 待完成步骤显示灰色。流程图下方提供「开始合作」CTA 按钮。

### 价格说明

价格说明区域以问答形式解答客户关于价格的常见疑问。常见问题包括：价格是如何计算的、项目周期如何预估、是否可以按阶段付款、是否支持维护期服务、如何处理需求变更导致的费用调整等。每个问题对应一段简洁的回答（50 至 100 字），帮助客户建立合理预期。问题以手风琴组件折叠显示，展开查看答案。

### 客户见证

客户见证区域展示合作客户的正面评价和合作成果。见证内容包括客户头像（或公司 Logo）、客户名称、合作项目、评价内容和评分（可选）。见证卡片采用横向滚动布局，移动端支持触摸滑动。见证数量控制在 3 至 5 条，优先展示与目标客户类型匹配的案例。

## 界面需求

### 视觉规范

服务定价模块整体采用与首页一致的深色主题。服务卡片背景色使用 #1e293b，边框 1 像素 #334155，圆角 16 卡片宽度在桌面端限制为 320px 至 380px，保持网格对齐。服务图标区域使用圆形或圆角方形背景，填充主色调的浅色变体（如 #f97316 的 20% 透明度）。

价格文字使用橙色或蓝色突出显示，字号 24px 至 32px，单位使用「元」或「¥」。价格说明使用 12px 灰色文字，位置紧跟价格区间下方。合作流程时间线使用水平线条连接节点，已完成节点使用绿色圆点，进行中节点使用橙色圆点，待完成节点使用灰色圆点。

### 响应式设计

桌面端（屏幕宽度 ≥ 1024px）服务卡片显示三列或四列，横向排列。服务分类标签横向展开，合作流程时间线水平展示，客户见证卡片三列并排。平板端（屏幕宽度 768px 至 1023px）服务卡片显示两列，分类标签部分折叠为下拉菜单，时间线在移动端自动切换流程为垂直布局。移动端（屏幕宽度 < 768px）服务卡片单列显示，分类标签水平滚动，流程时间线垂直展示，客户见证卡片单列滚动。

### 动画效果

页面加载时服务卡片采用网格交错淡入动画，每张卡片延迟 100 毫秒依次出现。服务分类切换时当前卡片淡出、新卡片淡入。卡片悬停时有上浮和阴影加深效果，交互结束后恢复原状。合作流程步骤在滚动进入视口时有从左到右的进度动画。客户见证卡片支持自动轮播，每 5 秒切换一张，轮播时有滑动淡入淡出效果。

### 流程数据

合作流程数据存储于 data/process.json，以数组形式组织，每项包含 title、description、icon 和 estimatedDuration 字段。

### 见证数据

客户见证数据存储于 data/testimonials.json，以数组形式组织，每项包含 clientName、clientTitle、company、avatar、project、content 和 rating 字段。

## 验收标准

1. 服务卡片在桌面端三列或四列显示、平板端两列、移动端单列，布局正确无溢出。
2. 价格区间显示正确，单位、范围和说明文字完整清晰。
3. 服务分类切换功能正常，动画流畅，数据对应准确。
4. 合作流程时间线在桌面端水平展示、移动端垂直展示，步骤状态正确。
5. 页面 Lighthouse 性能评分：性能 ≥ 90，可访问性 ≥ 90，最佳实践 ≥ 90。
6. 所有交互元素有点击反馈，动画流畅不卡顿。
7. 数据文件格式校验通过，服务卡片内容完整无缺失。
8. 响应式布局在 320px 至 1920px 屏幕宽度范围内正确适配。
9. 咨询按钮点击后正确跳转到联系表单或打开咨询模态框。
