# 首页模块规范

## 概述

首页模块是个人作品展示网站的核心入口页面，承担着第一印象建立、信息传递和用户引导的关键职责。该模块需要在一个页面内清晰传达工程师的专业身份、核心技术能力、可承接服务范围以及如何联系。页面设计应突出 Golang 后端开发的专业性，同时保持视觉简洁和交互流畅，让访客在 30 秒内快速理解服务价值并产生进一步了解的意愿。

首页采用单屏滚动布局，包含四个核心区域：顶部导航栏、个人简介区、技能展示区和服务概览区。各区域之间通过视觉分隔和留白实现自然过渡，确保信息层次清晰。页面整体风格应体现技术专业性，避免过度装饰，色彩以深色调为主，点缀橙色或蓝色等具有科技感的配色元素。

## 功能需求

### 导航栏

导航栏固定在页面顶部，包含 Logo、个人头像、导航链接和社交图标四个组件。Logo 区域显示工程师的名字或品牌标识，采用简洁的字体设计。导航链接包括「首页」「作品」「服务」「联系」四个锚点，点击后平滑滚动到对应区域。社交图标区域显示 GitHub、掘金、微信等平台链接，每个图标应有悬停动效。导航栏背景在滚动时保持固定，背景色可随滚动深度微调透明度，增强视觉层次感。

### 个人简介区

个人简介区占满首屏高度，采用左文右图或上文下图的布局结构。左侧或上方展示个人头像，圆形或圆角矩形裁切，尺寸建议 120×120 像素。文字区域包含主标题（姓名或品牌名称）、副标题（职位或专业方向）和一段 80 至 120 字的个人简介。简介内容应突出 Golang 开发经验、行业背景和核心价值主张。底部可添加「在线接单」状态指示器，显示当前是否可接新项目。CTA 按钮引导访客查看作品集或直接联系，按钮样式应醒目且符合整体配色风格。

### 技能展示区

技能展示区采用标签云或网格布局展示技术能力。技能分类包括后端开发（Go、gRPC、MySQL、Redis、Kafka）、微服务（Docker、Kubernetes、Istio、Prometheus）、架构设计（DDD、CQRS、Event Sourcing）和其他技能（Linux、Git、CI/CD）。每个技能标签应有不同的视觉权重，熟练程度高的技能使用更显眼的颜色或更大的字号。标签支持悬停交互，显示技能相关的项目经验或使用时长。技能区可添加「查看技术博客」链接，引导访客阅读技术文章。

### 服务概览区

服务概览区简要列出可承接的项目类型和服务优势。服务类型以图标加简短描述的形式呈现，每个服务卡片包含服务名称、简要说明和参考价格区间。服务项目包括「Web 后端开发」「API 接口设计」「微服务架构」「代码重构优化」「技术咨询」五项。优势区域使用三至四个关键词概括合作价值，如「高效沟通」「代码规范」「按时交付」「技术支持」。该区域底部设置「了解更多」按钮，滚动到服务详情页面。

## 界面需求

### 视觉规范

页面采用深色主题，背景色使用 #0f172a（深蓝灰）或 #1a1a2e（深紫黑）。文字颜色区分层级，主标题使用 #f8fafc（极白），正文使用 #cbd5e1（浅灰蓝），辅助文字使用 #64748b（深灰）。主色调为橙色（#f97316）或蓝色（#3b82f6），用于按钮、链接和强调元素。字体选用系统默认无衬线字体，中文使用「思源黑体」或「苹方」，英文使用 Inter 或 Roboto。字号规范：主标题 48px，副标题 24px，正文 16px，辅助文字 14px。

### 响应式设计

桌面端（屏幕宽度 ≥ 1024px）采用多列布局，导航栏水平展开，技能区使用网格排列。平板端（屏幕宽度 768px 至 1023px）调整列数为两列，导航栏保持水平但简化图标。移动端（屏幕宽度 < 768px）采用单列布局，导航栏变为汉堡菜单，技能区使用单列或两列小网格。所有断点的交互元素点击区域不小于 44×44 像素，确保触摸操作友好。

### 动画效果

页面加载时各区域采用淡入动画依次呈现，间隔 200 毫秒。导航链接悬停时有颜色过渡和下划线展开效果。技能标签悬停时放大 1.05 倍并显示轻微阴影。CTA 按钮悬停时背景色加深并有上浮效果。页面滚动时导航栏保持固定，可添加轻微的视差滚动效果增强层次感。动画总时长控制在 500 毫秒以内，避免拖沓感。

## 数据需求

### 个人信息数据

个人数据存储于 data/personal.json，包含 name（姓名或品牌名）、title（职位描述）、bio（个人简介，80 至 120 字）、avatar（头像图片路径）、status（接单状态，available/busy/暂停接单）和 contact（联系方式对象，包含 email、wechat、github、juejin 等字段）。数据示例：

```json
{
  "name": "张工",
  "title": "Golang 后端架构师",
  "bio": "5 年 Go 开发经验，擅长高并发分布式系统设计。曾主导多个千万级日活项目的后端架构，熟悉微服务架构、云原生部署和性能优化。",
  "avatar": "/images/avatar.jpg",
  "status": "available",
  "contact": {
    "email": "zhang@example.com",
    "wechat": "golang-dev",
    "github": "github.com/zhangdev",
    "juejin": "juejin.cn/user/zhangdev"
  }
}
```

### 技能数据

技能数据存储于 data/skills.json，以数组形式组织，每项包含 name（技能名称）、category（分类）、level（熟练度 1 至 5）和 relatedProjects（关联项目 ID 数组）。技能分类与作品集分类对应，支持筛选功能。

### 服务数据

服务数据存储于 data/services.json，以数组形式组织，每项包含 name（服务名称）、description（简短描述）、priceRange（价格区间）和 features（功能特点数组）。

## 验收标准

1. 页面在主流浏览器（Chrome、Firefox、Safari、Edge）最新版本中正常显示，交互功能可用。
2. 桌面端、平板端、移动端的响应式布局正确，无内容溢出或截断。
3. 页面 Lighthouse 性能评分：性能 ≥ 90，可访问性 ≥ 90，最佳实践 ≥ 90，SEO ≥ 90。
4. 图片资源使用 WebP 格式，页面总资源大小不超过 500KB，首屏加载时间不超过 2 秒。
5. 导航栏在滚动时保持固定，锚点滚动平滑跳转到对应区域。
6. 所有交互元素有清晰的悬停和点击反馈，动画流畅无卡顿。
7. 数据文件修改后页面内容自动更新，无需重新编译构建。
