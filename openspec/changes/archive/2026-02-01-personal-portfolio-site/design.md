# 个人作品展示网站技术设计

## 背景与现状

当前 Golang 工程师缺乏专业的线上展示平台，难以在闲鱼、小红书等渠道获取高质量外包订单。现有社交平台虽然可以发布信息，但缺乏系统化的作品展示和专业的品牌形象。潜在客户无法快速评估工程师的技术能力和项目经验，导致沟通成本高、转化率低。

本项目旨在构建一个轻量级、高性能的静态网站，作为个人品牌的核心载体。该网站将展示工程师的专业背景、技术栈、过往项目和接单信息，帮助潜在客户快速建立信任并促成合作。

## 目标与范围

**核心目标**：创建一个视觉专业、交互流畅的个人作品展示网站，实现以下功能：

- 清晰展示个人技术能力和专业背景
- 以卡片形式呈现过往项目作品，支持分类筛选
- 明确列出可承接的服务类型和价格区间
- 提供便捷的联系方式和在线需求提交功能

**非目标范围**：

- 不构建复杂的后端服务或用户认证系统
- 不实现博客系统或动态内容管理功能
- 不集成在线支付或订单管理系统
- 不支持多语言或国际化

## 技术决策

**静态站点生成方案**：采用 HTML + CSS + JavaScript 原生技术栈，不使用 React、Vue 等前端框架。决策依据如下：

- 网站内容以静态展示为主，无需复杂的状态管理和组件复用
- 原生技术可大幅减少构建复杂度，降低维护成本
- 有利于后续部署到 GitHub Pages、Netlify 等免费托管平台
- 减少首屏加载时间，提升访问体验

替代方案考量：使用 Hugo 或 Astro 等静态站点生成器可以提升开发效率，但会增加构建流程复杂度。对于单页面站点，原生开发更加轻量灵活。

**样式方案**：采用 Tailwind CSS 框架进行样式开发。Tailwind 提供了丰富的原子化 CSS 类，可快速构建响应式布局，同时保持最终产物体积小巧。通过 Tailwind 的配置可以自定义设计系统，确保视觉风格的一致性。

替代方案考量：使用 CSS Modules 或 Styled Components 会产生更多的样式代码，且响应式设计需要额外处理。Tailwind 的工具类集合更适合快速原型开发。

**项目数据管理**：使用 TypeScript 数据文件存储项目信息，通过 JSON 格式定义每个项目的标题、描述、技术栈、封面图片、项目链接等字段。页面渲染时动态加载这些数据，实现内容与代码的分离，便于后续维护和更新。

替代方案考量：使用 CMS 系统（如 Strapi、Contentful）虽然提供可视化编辑，但增加了部署和维护成本。对于个人作品网站，手动维护数据文件足够简单且高效。

**交互实现**：核心交互功能（平滑滚动、卡片悬停效果、分类筛选、模态框展开）使用原生 JavaScript 实现。动画效果采用 CSS transition 和 CSS Animation，减少 JavaScript 执行开销。

## 风险与权衡

**视觉设计风险**：作为技术型展示网站，视觉设计直接影响专业形象。潜在风险包括：排版混乱、色彩搭配不当、响应式布局在移动端表现不佳。

缓解措施：参考同类型优秀作品集网站的设计规范，采用清晰的视觉层次和一致的配色方案。使用 CSS Grid 和 Flexbox 实现可靠的响应式布局，在多种设备和屏幕尺寸下进行测试验证。

**内容维护风险**：项目展示需要持续更新，手动修改 HTML 文件可能造成遗漏或格式不一致。

缓解措施：将项目数据集中存储在独立的数据文件中，制定标准化的数据格式规范。添加简单的数据校验机制，确保新增项目信息的完整性和格式正确性。

**SEO 优化风险**：静态网站的 SEO 表现高度依赖页面结构和元信息配置。

缓解措施：正确设置页面标题、描述和 Open Graph 标签。为每个项目详情页生成唯一的 URL 和结构化数据标记。确保网站加载速度和移动端适配符合搜索引擎排名因素。
