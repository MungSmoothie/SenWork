# SenWork 项目规范提案

## 背景与现状

SenWork 是一个全栈个人作品集网站，用于展示 Golang 后端工程师的技能、项目和经验，帮助获取外包项目机会。当前项目已实现基础功能，包含 Vue 3 前端和 Go + Gin 后端。

**现有技术栈**:
- **前端**: Vue 3 + TypeScript + Vite
- **后端**: Go + Gin 框架
- **API**: RESTful 风格，5 个核心端点
- **数据**: 静态数据存储在 Go 代码中

**已实现功能**:
- 首页英雄区域展示
- 关于页面（个人简介、技能进度条、工作经历时间线）
- 项目作品展示
- 联系表单
- 响应式导航栏

## 规范范围

本规范定义 SenWork 项目的完整技术规范，包含两个子规范：

1. **senwork-api**: 后端 API 规范
   - API 端点定义
   - 请求/响应格式
   - 数据模型
   - 错误处理

2. **senwork-frontend**: 前端规范
   - 组件结构
   - 类型定义
   - API 调用层
   - 样式规范

## 新增能力

### senwork-api 能力

- **API 端点标准化**: 明确定义 5 个核心端点的请求响应格式
- **数据模型规范**: Go 数据结构与 TypeScript 类型定义保持一致
- **错误处理规范**: 统一的错误响应格式

### senwork-frontend 能力

- **组件架构**: 规范组件目录结构和命名约定
- **类型安全**: 完整的 TypeScript 类型定义
- **API 集成**: 封装统一的 API 调用层
- **响应式设计**: 明确的断点和布局规范

## 影响范围

### 后端影响

- 新增 API 文档（可使用 Swagger/OpenAPI）
- 数据验证中间件（可选）
- API 版本控制策略（未来扩展）

### 前端影响

- 新增组件设计规范
- 统一样式变量和主题系统
- 性能优化策略
- 无障碍访问规范

## 实施计划

### 第一阶段：规范定义

1. 完成 API 规范文档（已完成）
2. 完成前端规范文档（已完成）
3. 建立规范与代码的映射关系

### 第二阶段：代码同步

1. 更新后端 handler 类型注释
2. 更新前端类型定义与后端保持一致
3. 添加 API 文档生成配置

### 第三阶段：质量保障

1. 添加 API 端点测试
2. 建立类型检查 CI 流程
3. 文档维护和更新机制

## 扩展能力

### 短期扩展

- **数据库集成**: 从静态数据迁移至 PostgreSQL
- **API 文档**: 使用 swag 生成 Swagger 文档
- **缓存层**: 添加 Redis 缓存

### 长期扩展

- **用户系统**: 添加认证和授权
- **CMS 后台**: 管理项目、技能的动态更新
- **评论系统**: 项目评论和反馈功能
- **分析统计**: 访问量统计和用户行为分析

## 结论

本规范为 SenWork 项目建立了完整的技术标准，确保前后端代码的一致性和可维护性。通过规范化的 API 定义和前端组件架构，为后续功能扩展和团队协作奠定基础。
